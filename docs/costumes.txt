Costume resource format:
24 pairs of offset,length into the resource (not counting the header)
each pair consists of one set of frames for the actor (standing, walking, talking, etc. in each direction)
Non-terminated data follows (overall length can be determined from header)
This data points to frame numbers within the appropriate sections of resources DD9D/DD7D
(base addresses in said resources determined from resource DD95)

The actual frameset groups are as follows:
* Walking
* Standing
* ??? [same as standing]
* ??? [mouth open]
* ??? [same as standing]
* Talking

Each frameset group contains 4 framesets, one for each direction:
* Facing left
* Facing right (flipped)
* Facing forward
* Facing backward

resource DD95 is a list of 16-bit pointers into resources DD7D and DD9D (multiply by 2)
The last byte is apparently unused
0000 0011 0022 0033 0044 0055 0066 0077
0088 0099 00AA 00BB 00BD 00CC 00DD 00E1
00F2 00F8 00FC 00FD 00FE 0100 0104 0113 15

the engine uses the following lookup table to translate the costume resource number into an offset into this table:

00 03 01 06 08
02 00 07 0C 04
09 0A 12 0B 14
0D 11 0F 0E 10
17 00 01 05 16

resource DD75 is packed NES sprite data, 3 bytes per sprite:
fYYYYYYY,tttttttt,XXXXXXpp
f = horizontal flip
Y-coordinate = YYYYYYY
X-coordinate = XXXXXX
X and Y coordinates are signed, relative to the location of the actor
(too much to list here)

There are 2 separate sets of costume graphics resources. They are selected using a specific opcode.
